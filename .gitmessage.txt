# ----------------------------
# <type>: <subject>
#
# 타입(type):
# feat     새로운 기능 추가
# fix      버그 수정
# docs     문서 변경
# refactor 코드 리팩토링 (기능 변화 없음)
# chore    빌드/환경 설정/패키지 관리 등 기타 변경
# test     테스트 코드 추가/수정
# ----------------------------

# 예시:
# feat: 로그인 기능 추가
# fix: 사용자 조회 API 버그 수정
# docs: README 설치 방법 추가
# refactor: 중복된 유틸 함수 제거
# chore: eslint 설정 업데이트
# test: 회원가입 서비스 단위 테스트 추가
# ----------------------------

# subject는 한 줄 요약 (마침표 X, 소문자 시작 권장)
# body (선택): 변경 이유, 상세 설명
# footer (선택): 이슈 번호, BREAKING CHANGE 등
